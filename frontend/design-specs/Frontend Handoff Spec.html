§5 · Responsive Behavior

Responsive

Desktop is primary. Breakpoints defined for known fallbacks. Mobile phones are out of scope — polygon precision isn't viable under ~640 px.
desktop-xl ≥ 1440 px
Target layout · 70 / 30 split.
  • Feed column: 1fr, min 900 px. Alerts column: fixed 360 px.
  • App header: 64 px · padding-x 24.
  • Feed panel: 4:3 aspect, padding 20 around. Controls strip below.
  • Alert detail drawer: 440 px, overlays alerts column.
desktop 1200 – 1439 px
Holds 70 / 30.
  • Alerts column shrinks to 320 px min.
  • Feed-badge labels drop ("FPS 26.4" → "26.4" with mono label color cue).
  • Controls strip wraps to 2 rows if < 1260 px.
tablet 900 – 1199 px
Alerts collapse to drawer.
  • Feed takes full width. Alerts hidden behind a Alerts · 24 button in the header.
  • Drawer opens from right at 360 px width. Esc or scrim-click dismisses.
  • New-alert tint still plays; badge on the toggle shows unread count.
mobile/tablet-s 640 – 899 px
Stack vertically.
  • Order: header · feed · controls · alerts list (full-width).
  • Polygon draw still works with touch. Close Zone button enlarged to 44 px tap target.
  • Floating badges overflow hidden; status moves to header only.
out of scope < 640 px
Block with an informational screen.
  • "NeuraWatch requires a viewport of at least 640 px. Please use a larger screen."
  • No partial UI — polygon precision isn't viable and the bbox-to-zone math depends on readable geometry.

§6 · WebSocket Protocol

Data contract

Single WS protocol for webcam + upload modes. Server drives cadence. Client uses an in-flight boolean with 2 s watchdog. Frames are binary JPEG; detections are JSON.

6.2 · Client flow-control
In-flight boolean + watchdog
// services/wsClient.ts — single module-level singleton let inFlight = false; let lastSeq = 0; let seq = 0; let watchdog: ReturnType<typeof setTimeout>; function sendFrame(jpegBlob: Blob) { if (inFlight) return; // drop: server still has one inFlight = true; seq++; ws.send(JSON.stringify({ type: 'frame_meta', seq })); ws.send(jpegBlob); clearTimeout(watchdog); watchdog = setTimeout(() => { inFlight = false; // 2 s safety net dispatch({ type: 'ws/close', retry: true }); }, 2000); } ws.onmessage = (e) => { if (typeof e.data === 'string') { const msg = JSON.parse(e.data); if (msg.type === 'detection_result') { if (msg.seq <= lastSeq) return; // drop stale lastSeq = msg.seq; inFlight = false; clearTimeout(watchdog); dispatch({ type: 'ws/frame', payload: msg }); } } };
6.3 · Coordinate conversion
0–1 normalized ↔ canvas pixels
// All bboxes + polygon points are normalized 0–1 against 640×480 processed frame. // FE never stores pixel coords — always normalize on send, denormalize on render. function renderBbox(bbox: [number,number,number,number], canvasW: number, canvasH: number) { const [x1, y1, x2, y2] = bbox; ctx.strokeRect( x1 * canvasW, y1 * canvasH, (x2 - x1) * canvasW, (y2 - y1) * canvasH ); } function capturePoint(e: MouseEvent, canvasEl: HTMLCanvasElement): [number,number] { const r = canvasEl.getBoundingClientRect(); return [(e.clientX - r.left) / r.width, (e.clientY - r.top) / r.height]; }
Coordinate contract. If FE and BE disagree on coordinate space, alerts will fire in the wrong place — or never at all. Single source of truth: 0–1 against 640×480. Never store pixel coords in app state.
§7 · Accessibility
